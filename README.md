# PixelEra
PixelEra is a full-stack MERN application that provides AI-powered image background removal. It operates on a credit-based system, allowing users to sign up, receive free credits to try the service, and purchase more credits using the Cashfree payment gateway.

üîó **[Live Demo](https://pixel-era.vercel.app)**

## Core Features
- **Full-Stack Architecture**:
    - Frontend: A modern, responsive UI built with React, Vite, and Tailwind CSS.
    - Backend: A Node.js/Express server that manages API requests, user data, and payment logic.
- **AI Background Removal**: Integrates with the Clipdrop API to perform fast and accurate background removal.
- **User Authentication**: Secure sign-up, sign-in, and user management handled by Clerk.
- **Credit-Based System**: Users receive 5 free credits upon signing up and can purchase additional credits through the ‚ÄúBuy Credits‚Äù page.
- **Payment Integration**: Built-in payment flow using Cashfree to create orders and handle payment confirmation webhooks.
- **Database**: Uses MongoDB (via Mongoose) to store user data, including clerkId and creditBalance.
- **Modern UI/UX**: Includes an interactive "before & after" image slider, a dark mode toggle, and a seamless file upload process.


## Getting Started

#### 1. Clone the repository
```bash
git clone https://github.com/ImSonu2030/PixelEra.git
cd pixelera
```

#### 2. Install dependencies
```bash
npm install
```

#### 3. Create a .env file and add your credentials
```bash
VITE_CLIPDROP_API_KEY=your_clipdrop_key
CLERK_PUBLISHABLE_KEY=your_clerk_key
CASHFREE_CLIENT_ID=your_cashfree_client_id
CASHFREE_SECRET_KEY=your_cashfree_secret
MONGO_URI=your_mongodb_connection_string
```

#### 4. Run the app
```bash
npm run dev
```

# PixelEra - Mutation Testing Project

## 1. Test Case Strategy
**Method:** Mutation Testing
**Goal:** To validate the robustness of the test suite by deliberately introducing faults ("mutants") into the server-side logic (Controllers & Middleware) and ensuring the test suite detects ("kills") them.

**Testing Levels:**
1.  **Unit Level:** Testing individual functions in isolation (e.g., `auth.test.js`, `image.test.js`).
2.  **Integration Level:** Testing API endpoints and database interactions (e.g., `payment.test.js` simulating Webhooks and Order Creation).

**Mutation Operators Covered:**
The test suite successfully kills mutants generated by:
1.  **Conditional Expression:** Mutating `if (creditBalance === 0)` logic in ImageController.
2.  **Equality Operator:** Mutating `payment_status === "SUCCESS"` in PaymentController.
3.  **Arithmetic Operator:** Mutating credit deduction math (`balance - 1`).
4.  **String Literal:** Mutating string values like "SUCCESS", API URLs, and Webhook event types.
5.  **Block Statement:** Removing entire blocks of code (e.g., removing `switch` cases in User Webhooks).

**Results:**
- **Overall Mutation Score:** ~64%
- **Middleware Coverage:** ~88%
- **Key Achievement:** Successfully mocked external dependencies (Cashfree, Svix, Clipdrop) to allow deterministic testing of integration logic.

## 2. Designed Test Cases
The test suite was designed using **Mocha, Chai, Sinon, and Supertest**.

| Component | Critical Logic Tested | Test Scenario |
| :--- | :--- | :--- |
| **Image Controller** | Credit Balance Check | User with 0 credits receives specific error message (Mutant killed: Equality Operator) |
| **Image Controller** | API Interaction | Verifies correct API Key and URL are sent to Clipdrop (Mutant killed: String Literal) |
| **Payment Controller** | Order Creation | Verifies strict parameter passing to Cashfree SDK (Mutant killed: Object Property) |
| **Payment Webhook** | Payment Status | Failed payments do NOT update user balance (Mutant killed: Conditional Expression) |
| **User Webhook** | Event Type Handling | `user.created` triggers DB insert; `user.deleted` triggers DB remove (Mutant killed: Switch Case Removal) |
| **Auth Middleware** | Token Validation | Requests without specific headers are rejected (Mutant killed: Boolean Literal) |

## 3. Tools Used
* **Stryker Mutator:** For running mutation tests and generating mutants.
* **Mocha:** Test runner.
* **Chai:** Assertion library.
* **Sinon:** For mocking external dependencies (Cashfree, Svix, MongoDB, File System).

## 4. Execution Instructions
1.  Navigate to the server directory: `cd server`
2.  Install dependencies: `npm install`
3.  Run standard tests: `npm test`
4.  Run mutation tests: `npm run test:mutation`
